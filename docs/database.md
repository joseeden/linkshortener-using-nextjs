# Database

## Overview

The database layer uses **Drizzle ORM** with a **Neon PostgreSQL** serverless database.

- The Drizzle client is exported from `db/index.tsx` as `db`.
- The schema is defined in `db/schema.ts`.
- Migrations are managed with `drizzle-kit` and configured in `drizzle.config.ts`.

---

## Rules

- **Drizzle ORM is the only way to query the database.** Do not use raw SQL strings or any other ORM.
- **All schema changes must go through `db/schema.ts`.** Do not mutate the database outside of migrations generated by `drizzle-kit`.
- **Never use `any` for column types or query results.** Use the inferred types exported from `db/schema.ts`.
- **All columns must be `notNull()`.** There are no optional fields.
- **There are no soft deletes.** Do not add `deletedAt` or any deletion flag to any table.

---

## Schema

### `links` table

Represents a single shortened link owned by a Clerk user.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | `integer` | primary key, generated always as identity | Auto-incrementing surrogate key |
| `userId` | `text` | not null | Clerk user ID of the owner |
| `url` | `text` | not null | The original destination URL |
| `slug` | `text` | not null, unique | The unique short code used to resolve the link |
| `createdAt` | `timestamp with time zone` | not null, default now | Set once on insert |
| `updatedAt` | `timestamp with time zone` | not null, default now, auto-updates | Updated automatically on every row update |

```ts
// db/schema.ts
import { integer, pgTable, text, timestamp } from 'drizzle-orm/pg-core';

export const links = pgTable('links', {
  id: integer('id').primaryKey().generatedAlwaysAsIdentity(),
  userId: text('user_id').notNull(),
  url: text('url').notNull(),
  slug: text('slug').notNull().unique(),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow().$onUpdate(() => new Date()),
});

export type Link = typeof links.$inferSelect;
export type NewLink = typeof links.$inferInsert;
```

## Inferred Types

Always use the inferred types from the schema instead of defining your own interfaces.

```ts
import { type Link, type NewLink } from '@/db/schema';

// Link — shape returned by SELECT queries
// NewLink — shape accepted for INSERT queries (id, createdAt, updatedAt are optional/generated)
```

---

## Querying

Import `db` from `@/db` and use Drizzle's query builder.

```ts
import { db } from '@/db';
import { links } from '@/db/schema';
import { eq } from 'drizzle-orm';

// Select all links for a user
const userLinks = await db.select().from(links).where(eq(links.userId, userId));

// Insert a new link
await db.insert(links).values({ userId, url, slug });

// Delete a link
await db.delete(links).where(eq(links.id, id));
```

## Migrations

Generate and apply migrations with `drizzle-kit`:

```bash
# Generate a migration from schema changes
npx drizzle-kit generate

# Apply pending migrations
npx drizzle-kit migrate
```

The output directory for migrations is `./drizzle` (configured in `drizzle.config.ts`).
